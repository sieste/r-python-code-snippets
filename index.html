NumPy Basics

Like R, python requires explicit library imports at the top of the script to add new functions. Unlike R, python does not have native support for vectors and matrices and needs to import NumPy, Python's core numerical library. Note: Python arrays are 0-indexed (first element is index 0, not 1).


# Create a vector
x <- c(1, 2, 3, 4, 5)

# Access elements (1-indexed)
x[1]  # Returns 1
x[2:4]  # Returns 2, 3, 4

# Element-wise operations work naturally
y <- x * 2
z <- x + 5

x[0]  # Returns 1 (not x[1])

NumPy Summary Statistics and Mathematical Functions

NumPy provides vectorized mathematical functions like np.sqrt(), np.sum(), np.mean(), etc. that work on arrays. Key difference: instead of na.rm=TRUE, use np.nansum(), np.nanmean(), etc. to ignore NaN values. Many functions accept axis parameter to compute along rows (axis=0) or columns (axis=1).


# Create a vector with missing values
x <- c(1, 2, 3, NA, 5)

# Basic math functions
sqrt(x)  # Element-wise square root
abs(x)   # Absolute value
log(x)   # Natural log

# Summary statistics
sum(x)   # Returns NA
sum(x, na.rm = TRUE)  # Returns 11

mean(x)  # Returns NA
mean(x, na.rm = TRUE)  # Returns 2.75

sd(x, na.rm = TRUE)  # Standard deviation
var(x, na.rm = TRUE)  # Variance
min(x, na.rm = TRUE)  # Minimum
max(x, na.rm = TRUE)  # Maximum

# On matrices/2D arrays
mat <- matrix(1:12, nrow = 3)
colSums(mat)  # Sum down columns
rowMeans(mat)  # Mean across rows
import numpy as np

# Create array with NaN (Python's missing value)
x = np.array([1, 2, 3, np.nan, 5])

# Basic math functions
np.sqrt(x)  # Element-wise square root
np.abs(x)   # Absolute value
np.log(x)   # Natural log

# Summary statistics
np.sum(x)  # Returns NaN
np.nansum(x)  # Returns 11.0 (ignores NaN)

np.mean(x)  # Returns NaN
np.nanmean(x)  # Returns 2.75 (like na.rm=TRUE)

np.nanstd(x)  # Standard deviation
np.nanvar(x)  # Variance
np.nanmin(x)  # Minimum
np.nanmax(x)  # Maximum

# On 2D arrays
mat = np.arange(1, 13).reshape(3, 4)
np.nansum(mat, axis=1)  # Sum across rows
np.nansum(mat, axis=0)  # Sum down columns
np.nanmean(mat, axis=1)  # Mean across rows

# GOTCHA: axis=0 is DOWN (columns), axis=1 is ACROSS (rows)
# GOTCHA: Use np.nan* versions to skip NaN values
# GOTCHA: Regular np.sum() returns NaN if any NaN present

NumPy Array Indexing and Slicing

NumPy indexing is 0-based and uses square brackets. Slicing syntax is [start:stop:step] where stop is exclusive. Critical gotcha: slices are views not copies—modifying a slice modifies the original array. Use .copy() if you need an independent copy. Boolean and fancy indexing create copies, but simple slices do not.

# Create a vector
x <- c(10, 20, 30, 40, 50)

# Indexing (1-indexed)
x[1]  # Returns 10
x[c(1, 3, 5)]  # Returns 10, 30, 50

# Slicing
x[1:3]  # Returns 10, 20, 30 (inclusive on both ends)
x[c(2:4)]  # Returns 20, 30, 40

# Negative indexing (exclude)
x[-1]  # Returns 20, 30, 40, 50
x[c(-1, -5)]  # Exclude first and last

# Logical indexing
x[x > 25]  # Returns 30, 40, 50

# Modify a subset
x[1:2] <- c(100, 200)
# x is now: 100, 200, 30, 40, 50

# Create a matrix
mat <- matrix(1:12, nrow = 3)
mat[1, ]  # First row
mat[, 2]  # Second column
mat[1:2, 2:3]  # Subset
import numpy as np

# Create an array
x = np.array([10, 20, 30, 40, 50])

# Indexing (0-indexed!)
x[0]  # Returns 10
x[[0, 2, 4]]  # Returns [10, 30, 50]

# Slicing [start:stop:step] (stop is exclusive!)
x[0:3]  # Returns [10, 20, 30] (not including 40)
x[1:4]  # Returns [20, 30, 40]

# Negative indexing (from end)
x[-1]  # Returns 50 (last element)
x[-3:]  # Returns [30, 40, 50] (last 3)

# Step parameter
x[::2]  # Every other element: [10, 30, 50]
x[1::2]  # Every other starting at index 1: [20, 40]
x[::-1]  # Reverse: [50, 40, 30, 20, 10]

# Logical indexing
x[x > 25]  # Returns [30, 40, 50]

# GOTCHA: Slices are VIEWS, not copies!
x_slice = x[1:4]  # This is a view
x_slice[0] = 999  # Modifies original x!
# x is now: [10, 999, 30, 40, 50]

# Use .copy() to make independent copy
x_copy = x[1:4].copy()
x_copy[0] = 999  # Does NOT modify x

# On 2D arrays
mat = np.arange(1, 13).reshape(3, 4)
mat[0, :]  # First row
mat[:, 1]  # Second column
mat[0:2, 1:3]  # Subset (rows 0-1, cols 1-2)

# GOTCHA: Slicing is 0-indexed and stop is exclusive
# GOTCHA: x[1:4] gets indices 1, 2, 3 (not 4!)
# GOTCHA: Slices are views—modifying them modifies original
# GOTCHA: Always use .copy() if you need independence

In Python/Matplotlib, create the grid upfront with plt.subplots() and access panels via ax[i, j] indexing. The Python approach is more explicit but requires planning your layout before plotting.

Stochastic Simulation: Temperature Time Series with Seasonality

NumPy's np.random module provides functions to generate random numbers for simulation. Common functions: np.random.normal() for Gaussian noise, np.random.uniform() for uniform random values, np.random.seed() for reproducibility. This example simulates a temperature time series with seasonal trend and random noise, then plots the results with both the noisy data and smoothed trend.

# Set seed for reproducibility
set.seed(42)

# Create time vector (days, 2 years)
days <- 1:730
t <- days / 365.25

# Simulate temperature: trend + seasonality + noise
trend <- 15 + 5 * t
seasonal <- 10 * sin(2 * pi * t)
noise <- rnorm(length(days), mean=0, sd=2)
temperature <- trend + seasonal + noise

# Plot
plot(days, temperature, type='l', col='gray', 
     main='Simulated Temperature Time Series',
     xlab='Days', ylab='Temperature (°C)',
     ylim=c(0, 35))

# Overlay smoothed trend
lines(days, trend + seasonal, col='red', lwd=2)

# Add legend
legend('topleft', c('Observed (with noise)', 'True signal'),
       col=c('gray', 'red'), lty=c(1, 1))

Functions and Indentation (Critical!)

Python uses indentation to define code blocks (not braces). This is grammatically required—incorrect indentation is a syntax error. Functions must return values explicitly. The first line after : must be indented consistently.

# Define a function
calculate_bonus <- function(salary, 
                             rate = 0.1) {
  bonus <- salary * rate
  return(bonus)
}

# Call the function with positional and named arguments
my_bonus <- calculate_bonus(50000)
my_bonus <- calculate_bonus(50000, 0.15)
my_bonus <- calculate_bonus(rate = 0.15, salary = 50000)

# Define a function
# CRITICAL: indentation defines the function block!
def calculate_bonus(salary, rate=0.1):
    bonus = salary * rate
    return bonus

# Call the function with positional and named arguments
my_bonus = calculate_bonus(50000)
my_bonus = calculate_bonus(50000, rate=0.15)
my_bonus = calculate_bonus(rate = 0.15, salary = 50000)

# GOTCHA: Indentation is REQUIRED syntax
# GOTCHA: Must explicitly return (otherwise function returns None)

Data Types: NumPy Arrays and Type Strictness

R coerces types flexibly (often silently). Python and NumPy are strict—type mismatches cause errors instead of silent coercion. Every NumPy array has a single dtype (e.g., int64, float64, object). Operations between incompatible types fail. You must explicitly convert using astype(), int(), float(), etc.

# R coerces flexibly
x <- "5"
y <- x + 3  # R converts "5" to numeric
# Result: 8

# Vectors coerce automatically
nums <- c("1", "2", "3")
mean(nums)  # Coerces to numeric with warning

# Check types
class(x)    # "character"
typeof(x)   # "character"
class(nums) # "character"

# Mixing types in vector
mixed <- c(1, 2, "three")
# All coerced to character: "1" "2" "three"
import numpy as np

# Python does NOT coerce
x = "5"
y = x + 3  # TypeError! Can't add str + int

# Must be explicit
y = int(x) + 3  # Converts to int first, result: 8

# NumPy arrays have a single dtype
nums = np.array(['1', '2', '3'])
nums.dtype  # dtype('U21') - Unicode string type

# Cannot compute mean on string array
np.mean(nums)  # TypeError!
# Must convert first
np.mean(nums.astype(int))  # Result: 2.0

# Check types
type(x)  # 
nums.dtype  # Shows array dtype
nums[0].dtype  # Shows element dtype

# Mixing types forces common dtype
mixed = np.array([1, 2, 'three'])
mixed.dtype  # dtype('U21') - all converted to string
# Result: ['1' '2' 'three']

# Creating array with specific dtype
floats = np.array([1, 2, 3], dtype=float)
floats.dtype  # dtype('float64')

# GOTCHA: No silent coercion—must convert explicitly
# GOTCHA: Each array has ONE dtype for all elements
# GOTCHA: String "5" is different from int 5
# GOTCHA: Operations fail fast rather than coercing
# GOTCHA: Mixing types forces lowest-common dtype (usually string)

Fitting and Analyzing Linear Regression Models

R's lm() provides integrated model fitting with built-in diagnostic tools. Python requires external libraries: statsmodels mimics R's functionality with formula interface and comprehensive diagnostics, while scikit-learn is more minimalist and focused on prediction. Statsmodels gives you p-values and confidence intervals; sklearn requires manual computation or alternative libraries.

# Set seed and simulate data
set.seed(42)
n <- 100
x <- rnorm(n, mean=50, sd=10)
y <- 10 + 1.5 * x + rnorm(n, mean=0, sd=15)

# Fit linear model
model <- lm(y ~ x)

# View results
summary(model)

# Extract key statistics
coef(model)         # Coefficients
confint(model)      # Confidence intervals
residuals(model)    # Residuals
fitted(model)       # Fitted values

# Model diagnostics
r_squared <- summary(model)$r.squared
adj_r_squared <- summary(model)$adj.r.squared
aic <- AIC(model)

# Prediction
new_x <- data.frame(x = c(40, 50, 60))
predict(model, newdata=new_x, 
        se.fit=TRUE, interval='confidence')
import numpy as np
import statsmodels.api as sm
from sklearn.linear_model import LinearRegression
from sklearn.metrics import r2_score, mean_squared_error

# Set seed and simulate data
np.random.seed(42)
n = 100
x = np.random.normal(loc=50, scale=10, size=n)
y = 10 + 1.5 * x + np.random.normal(loc=0, scale=15, size=n)

# ===== Using statsmodels (R-like) =====
# Add constant for intercept
X_sm = sm.add_constant(x)

# Fit model
model_sm = sm.OLS(y, X_sm).fit()

# View results (similar to R's summary)
print(model_sm.summary())

# Extract key statistics
params = model_sm.params  # Coefficients
conf_int = model_sm.conf_int()  # Confidence intervals
residuals = model_sm.resid  # Residuals
fitted = model_sm.fittedvalues  # Fitted values

r_squared = model_sm.rsquared
adj_r_squared = model_sm.rsquared_adj
aic = model_sm.aic
pvalues = model_sm.pvalues

# Prediction with confidence intervals
new_x_sm = sm.add_constant([40, 50, 60])
predictions_sm = model_sm.get_prediction(new_x_sm)
pred_summary = predictions_sm.summary_frame(alpha=0.05)
print(pred_summary)

# ===== Using scikit-learn (minimalist) =====

# Extract results
intercept = model_sk.intercept_
slope = model_sk.coef_[0]

r2 = r2_score(y, y_pred)
rmse = np.sqrt(mean_squared_error(y, y_pred))
residuals_sk = y - y_pred

# Prediction (no confidence intervals by default)
new_x_sk = np.array([40, 50, 60]).reshape(-1, 1)
predictions_sk = model_sk.predict(new_x_sk)

Filtering Rows - Boolean Indexing

R's filter() is intuitive, but Python's boolean indexing feels backwards at first. You create a boolean mask, then use it in square brackets. This is powerful once you get used to it, but requires different thinking than dplyr pipes.

library(dplyr)

# Simple filter
adults <- df %>%
  filter(age >= 30)

# Multiple conditions (AND)
high_earners <- df %>%
  filter(age >= 30 & salary > 55000)

# OR condition
subset <- df %>%
  filter(name == "Alice" | name == "Charlie")

# NOT condition
exclude <- df %>%
  filter(!(age < 30))
# Create boolean mask, then filter
# This is the Pythonic way 
mask = df['age'] >= 30
adults = df[mask]

# or do it in one line
adults = df[df['age'] >= 30]

# Multiple conditions: use & 
# parentheses around each condition
high_earners = df[
    (df['age'] >= 30) & (df['salary'] > 55000)
]

# OR condition: use | 
subset = df[
    (df['name'] == 'Alice') | (df['name'] == 'Charlie')
]

# NOT condition: use ~ (not '!')
exclude = df[~(df['age'] < 30)]

Handling Missing Data (NaN vs NA)

Python uses np.nan for missing values (not NA). The way missing data behaves in operations, grouping, and plotting differs subtly from R. Most pandas functions skip NaNs by default, but you need to be explicit about it sometimes.

# Create data with missing values
df <- data.frame(
  x = c(1, 2, NA, 4, 5),
  y = c(10, NA, 30, 40, 50)
)

# Check for missing
is.na(df$x)  # Returns logical vector

# Remove rows with any NA
df_clean <- na.omit(df)

# Mean (NA propagates by default)
mean(df$x)  # Returns NA
mean(df$x, na.rm = TRUE)  # Returns 3

# Drop missing in specific column
df_no_na_x <- df %>%
  filter(!is.na(x))
import numpy as np
import pandas as pd

df = pd.DataFrame({
    'x': [1, 2, np.nan, 4, 5],
    'y': [10, np.nan, 30, 40, 50]

# Check for missing
df['x'].isna()  # Returns boolean Series

# Remove rows with any NaN
df_clean = df.dropna()

df['x'].mean()  # Returns 3.0 (not NaN!)

# Drop missing in specific column
df_no_na_x = df[df['x'].notna()]

# (skips NaN automatically), but you must be aware

Indexing: loc vs iloc (Critical Difference)

This is a major gotcha for R users. Python has TWO indexing methods: loc (label-based) and iloc (position-based, 0-indexed). Confusing these is a common source of bugs. R doesn't make this distinction.

# R uses one indexing system
df <- data.frame(
  name = c("Alice", "Bob", "Charlie"),
  age = c(25, 30, 35)
)

# By position (1-indexed)
df[1, ]  # First row
df[, 2]  # Second column

# By name
df[, "age"]  # age column
df$age  # Same as above

# Logical indexing
df[df$age > 28, ]  # Rows where age > 28
df = pd.DataFrame({
    'name': ['Alice', 'Bob', 'Charlie'],
    'age': [25, 30, 35]
})

# iloc: position-based (0-indexed)
df.iloc[0, :]  # First row
df.iloc[:, 1]  # Second column (age)
df.iloc[0:2, 0:2]  # First 2 rows, first 2 cols

# loc: label-based (by row/column names)
df.loc[0, 'age']  # Access by row index and column name
df.loc[:, 'age']  # All rows, age column

# Boolean indexing (same as R)
df[df['age'] > 28]  # Rows where age > 28

# GOTCHA: iloc uses 0-indexing
# GOTCHA: iloc slice [0:2] means rows 0-1 (not 0-2)
# GOTCHA: loc uses row labels (usually 0,1,2...)
# which are DIFFERENT from row positions!

Data Types: Be Explicit About Object vs Value Types

R coerces types flexibly (often silently). Python is stricter and you must be more conscious of data types. String vs numeric operations behave very differently. Type mismatches cause errors instead of silent coercion.

y <- x + 3  # R converts "5" to numeric
# Result: 8

# Works across operations
df <- data.frame(
  id = c("1", "2", "3"),
  value = c(10, 20, 30)
)
mean(df$id)  # Coerces to numeric with warning

# Check types
class(x)
typeof(x)
# Python does NOT coerce
x = "5"
y = x + 3  # TypeError! Can't add str + int

# Must be explicit
y = int(x) + 3  # Converts to int first
